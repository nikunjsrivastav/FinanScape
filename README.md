# FinanScape - Financial Conversation Intelligence System

FinanScape is an AI-powered multilingual system that detects financial conversations, records audio, transcribes it, and converts unstructured discussions into structured financial insights.

**Important Note:** This project relies **entirely** on the **Groq API** for AI capabilities. **No local AI models** are used in this implementation. This ensures ultra-fast, scalable, and memory-efficient processing without requiring a beefy GPU on the host machine.

## Key Features

- **Audio Capture UI**: A premium, glassmorphism-styled React interface to record financial conversations via the browser.
- **Lightning Fast Transcription**: Audio is sent to the backend and processed via Groq's high-speed `whisper-large-v3-turbo` model.
- **Streaming Insights (SSE)**: As soon as the text is transcribed, the analysis (done by Groq's `llama-3.1-8b-instant`) is **streamed** back to the React frontend using Server-Sent Events (SSE). You can watch the Summary, Entities, and Actionable Insights load in real-time.
- **Database History**: All conversations, transcripts, and insights are saved in a local SQLite database for future review.

## Architecture & Approach

### 1. Frontend: React + Vite
- Created with `vite` and styled entirely with vanilla CSS and CSS variables. No bulky UI frameworks like Tailwind were used to maintain pure styling control and implement an elegant dark-theme glassmorphism aesthetic.
- Relies on the native `MediaRecorder` API to capture audio chunks.
- Uses `EventSource` to intercept SSE packets from the backend to animate text generation dynamically.

### 2. Backend: Flask (Python)
- A lightweight REST API built with Flask.
- `api/upload` endpoint: Receives the `.webm` audio from the browser, temporarily saves it, and sends the raw file to the Groq Whisper API for transcription.
- `api/stream_insights/<id>` endpoint: Yields streaming chunks of text generated by Groq's LLaMA model. It specifically prompts the model to return rigorous `[SUMMARY]`, `[ENTITIES]`, and `[INSIGHTS]` blocks.
- On stream completion, it commits the generated blocks to the SQLite `conversations.db` database.

### Why No Local AI?
While tools like local Whisper or local Llama.cpp are powerful, they require significant machine resources and GPU power to run efficiently (especially multilingual variations). 
By leveraging Groq, FinanScape provides near-instantaneous speech-to-text and LLM inferencing, making the application feel snappy and premium without needing specialized hardware.

## How to Run Locally

### Prerequisites
- Node.js (v18+)
- Python (v3.10+)
- A Groq API Key

### 1. Backend Setup
1. Open a terminal in the `backend/` directory.
   ```bash
   cd backend
   ```
2. Create and activate a python virtual environment:
   ```bash
   python -m venv venv
   # Windows
   .\venv\Scripts\activate
   # Mac/Linux
   source venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Create a `.env` file inside `backend/` and add your Groq API Key:
   ```env
   GROQ_API_KEY=your_actual_key_here
   ```
5. Run the Flask API:
   ```bash
   python app.py
   ```
   *The server will run on http://localhost:5000*

### 2. Frontend Setup
1. Open a new terminal in the `frontend/` directory.
   ```bash
   cd frontend
   ```
2. Install the node modules:
   ```bash
   npm install
   ```
3. Start the Vite development server:
   ```bash
   npm run dev
   ```
   *The frontend will run on http://localhost:5173*

## Usage
1. Open the frontend URL in your browser locally.
2. Allow microphone permissions.
3. Click the microphone icon and speak organically about your financial planning (e.g., "I am thinking of starting a new SIP of 5000 rupees and taking a loan for a car").
4. Click stop. The application will immediately utilize Groq Whisper and stream the resultant LLM insights back to your dashboard in real-time.
